# M1：知识库与文件管理 — 详细功能需求

> 模块编号：M1  
> 所属项目：Cognee-Code AI 辅助开发平台  
> 版本：v1.0  

---

## 1. 模块概述

知识库与文件管理是整个平台的数据基础层。负责知识数据的导入、组织、存储、检索和删除。核心概念包括：
- **数据集（Dataset）**：知识库的顶层组织单位，按主题/项目分类管理知识
- **数据项（Data）**：数据集中的基本内容单元，代表一个文件或一段文本
- **知识搜索**：基于已构建的知识图谱和向量索引的多模式搜索

---

## 2. 数据集（Dataset）管理

数据集是知识库的顶层组织单位。每个数据集归属于一个所有者用户，并可通过权限系统共享给其他用户。数据集内包含多个数据项（文件/文本），是知识处理（Cognify）和搜索的基本单元。

### 2.1 创建数据集

**功能描述：** 用户可以创建一个新的知识数据集来组织不同主题或项目的知识数据。

**详细需求：**
- 用户通过名称创建数据集
- 若当前用户已拥有同名数据集，直接返回已有数据集，不重复创建
- 新创建的数据集自动授予当前用户全部权限：read（读取）、write（写入）、share（共享）、delete（删除）
- 数据集包含以下属性：
  - ID（UUID，自动生成）
  - 名称（用户指定）
  - 所有者 ID（创建用户）
  - 租户 ID（多租户隔离，可为空）
  - 创建时间
  - 更新时间
- 数据集创建时不包含任何数据，数据通过「添加数据」功能导入

**前端展示需求：**
- 提供创建数据集的表单界面（仅需输入名称）
- 创建成功后刷新数据集列表

### 2.2 查看数据集列表

**功能描述：** 用户可以查看自己有权访问的所有数据集。

**详细需求：**
- 列出当前用户具有 read 权限的所有数据集
- 每个数据集展示信息：
  - 数据集名称
  - 数据集 ID
  - 创建时间
  - 最后更新时间
  - 所有者 ID
- 支持区分「我创建的」与「他人共享给我的」数据集

**前端展示需求：**
- 数据集列表页，以卡片或表格形式展示
- 展示数据集名称、创建时间、数据量概览
- 支持点击进入数据集详情

### 2.3 删除数据集

**功能描述：** 用户可以删除整个数据集，包括其包含的所有数据项。

**详细需求：**
- 通过数据集 ID 执行删除
- 用户必须对该数据集具备 delete 权限
- 删除后该数据集及其下所有数据项被永久移除
- 若数据项仅属于该数据集，则一并从数据表中删除
- 若数据项同时属于其他数据集，则仅解除关联关系

**前端展示需求：**
- 删除前弹出确认对话框，提示「此操作不可恢复」
- 删除成功后从列表中移除

### 2.4 查看数据集处理状态

**功能描述：** 用户可以查看数据集的知识图谱构建处理状态。

**详细需求：**
- 查询指定数据集在 cognify_pipeline 中的执行状态
- 支持批量查询多个数据集的状态
- 状态值包括：
  - **pending**：排队等待处理
  - **running**：正在处理中
  - **completed**：处理完成
  - **failed**：处理失败
- 用户需对目标数据集具有 read 权限

**前端展示需求：**
- 在数据集列表或详情页显示处理状态标签（颜色区分）
- 处理中的数据集显示进度指示
- 失败状态提供错误信息提示

### 2.5 本地目录扫描发现数据集

**功能描述：** 扫描本地文件目录，自动发现可作为数据集的子目录。

**详细需求：**
- 输入一个目录路径
- 返回该目录下可被识别为数据集的子目录名称列表

## 3. 数据（Data/文件）管理

数据（Data）是知识库中的基本内容单元，代表一个文件或一段文本。每个数据项关联到一个或多个数据集。系统在添加数据时会自动提取文件元信息、生成内容哈希用于去重，并存储到本地/云端文件系统。

### 3.1 添加数据

**功能描述：** 用户向指定数据集中导入知识数据（文件、文本、URL 等）。

**支持的输入类型：**
- **文本字符串**：直接输入的文本内容
- **本地文件**：
  - 绝对路径：`/path/to/document.pdf`
  - File URL：`file:///path/to/document.pdf` 或 `file://relative/path.txt`
  - S3 路径：`s3://bucket-name/path/to/file.pdf`
- **文件上传**：通过表单上传的文件流（BinaryIO）
- **HTTP URL**：远程网页内容（需开启 ALLOW_HTTP_REQUESTS）
- **GitHub 仓库 URL**：自动克隆并处理整个仓库
- **批量添加**：以上类型的混合列表

**支持的文件格式：**

| 分类 | 格式 | 处理方式 |
|------|------|---------|
| 文本文件 | .txt、.md、.csv | 直接提取文本 |
| PDF 文档 | .pdf | PDF 解析提取 |
| 图片文件 | .png、.jpg、.jpeg | OCR/视觉模型提取文本 |
| 音频文件 | .mp3、.wav | 语音转录为文本 |
| 代码文件 | .py、.js、.ts 等 | 解析代码结构和内容 |
| Office 文档 | .docx、.pptx | 文档解析提取 |

**处理流程：**
1. **数据解析**：解析文件路径、验证可访问性
2. **内容提取**：从各种文件格式中提取文本内容（使用对应的 loader 引擎）
3. **去重检测**：基于内容哈希 + 用户 ID 生成唯一数据 ID，自动跳过已存在的相同内容
4. **文本转换**：将所有内容统一转换为文本文件存储到 Cognee 数据存储
5. **元数据记录**：记录文件名、扩展名、MIME 类型、文件大小、内容哈希、原始存储路径等
6. **数据集关联**：将数据项关联到目标数据集
7. **权限分配**：为用户授予数据集的 read/write/delete/share 权限

**数据项属性：**
- ID（基于内容哈希+用户ID自动生成的 UUID）
- 名称（原始文件名或自动生成）
- 标签 label（用户自定义标注）
- 扩展名、MIME 类型（存储文件的）
- 原始扩展名、原始 MIME 类型（原始文件的）
- 原始数据存储位置（raw_data_location）
- 原始文件存储位置（original_data_location）
- 内容哈希 content_hash（原始文件，用于去重）
- 原始内容哈希 raw_content_hash（存储文件）
- 所有者 ID、租户 ID
- node_set（节点集标签，JSON 数组）
- pipeline_status（管道处理状态，JSON 对象）
- token_count（文本 token 数量）
- 文件大小 data_size（字节）
- loader_engine（使用的加载引擎名称）
- 外部元数据 external_metadata（JSON，可附加自定义信息）
- 创建时间、更新时间、最后访问时间

**其他详细需求：**
- 数据必须归属到一个数据集（通过 datasetName 或 datasetId 指定）
- 支持 node_set 标签分组，用于知识图谱中的数据组织和访问控制
- 支持自定义标签（label）标注数据项（通过 DataItem 包装器）
- 支持自定义 loader 引擎选择（preferred_loaders）
- 支持增量加载模式（默认开启），避免重复处理相同内容
- 支持 Web 抓取规则配置（extraction_rules，CSS 选择器 / XPath）
- 支持 Tavily API 配置（用于 URL 内容提取）
- 支持每批处理数量配置（data_per_batch，默认 20）

**前端展示需求：**
- 提供文件拖拽上传区域
- 提供 URL 输入框（支持 HTTP URL、GitHub URL）
- 提供文本直接输入框
- 上传时可选择或创建目标数据集
- 上传时可设置 node_set 标签
- 展示上传进度和结果

### 3.2 查看数据列表

**功能描述：** 查看某个数据集下的所有数据项（文件/文本）。

**详细需求：**
- 按数据集 ID 查询其下所有数据项
- 需对该数据集具有 read 权限
- 每个数据项展示信息：
  - 数据 ID
  - 名称
  - 标签
  - 扩展名
  - MIME 类型
  - 原始文件存储位置
  - 所属数据集 ID
  - 创建时间
  - 更新时间

**前端展示需求：**
- 数据集详情页内展示数据列表（表格形式）
- 支持按名称、类型、时间排序
- 支持文件类型图标展示
- 每行提供操作按钮（下载、删除、更新）

### 3.3 更新数据

**功能描述：** 更新已有数据项的内容（替换文件），自动重新构建知识图谱。

**详细需求：**
- 输入参数：目标数据 ID + 目标数据集 ID + 新数据内容
- 更新流程（原子操作）：
  1. **删除旧数据**：删除旧数据及其在图谱中的子图（调用 delete）
  2. **添加新数据**：将新数据添加到同一数据集（调用 add）
  3. **重建图谱**：自动触发 Cognify 重建知识图谱（调用 cognify）
- 支持的输入类型与「添加数据」一致
- 支持 node_set 标签重新设置
- 返回更新后的管道运行信息

**前端展示需求：**
- 数据项详情页提供「替换文件」按钮
- 显示更新进度（因为会触发 Cognify）

### 3.4 删除数据

**功能描述：** 从数据集中删除特定的数据项。

**入口一：按数据 ID 删除（支持软/硬模式）**

输入参数：data_id + dataset_id + mode

用户需对数据集具有 delete 权限。

**删除流程：**

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 图谱子图删除 | 按固定顺序维护图完整性 |
| 1.1 | 删除孤立实体 | orphan_entities |
| 1.2 | 删除孤立实体类型 | orphan_types |
| 1.3 | 删除 made_from 节点 | 摘要等衍生节点 |
| 1.4 | 删除文档 chunk | 文本分块 |
| 1.5 | 删除文档节点 | 文档本体 |
| 2 | 向量库删除 | 从所有向量集合中删除对应嵌入向量 |
| 3 | 关系账本更新 | 标记 graph_relationship_ledger 的 deleted_at |
| 4 | 关系数据库删除 | 解除数据-数据集关联，必要时删除数据记录 |

**两种删除模式：**
- **软删除（soft）**：仅删除文档直接相关节点，保留可能被其他文档共享的实体节点
- **硬删除（hard）**：额外删除度为 1 的孤立 Entity 和 EntityType 节点

**返回信息：**
- 操作状态
- 各类节点删除数量统计
- 已删除节点 ID 列表
- 文档 ID 和数据集信息

**入口二：从数据集中移除数据项**

输入参数：dataset_id + data_id

仅解除数据与数据集的关联关系，不执行图谱级别的删除。

**前端展示需求：**
- 数据项行提供删除按钮
- 删除前弹出确认对话框
- 提供软删除/硬删除选项说明
- 删除后刷新列表

### 3.5 下载原始数据

**功能描述：** 下载数据项的原始文件。

**详细需求：**
- 支持两种存储来源：
  - **本地文件系统**：直接通过 FileResponse 返回文件
  - **S3 存储**：通过 StreamingResponse 流式下载（1MB chunk），支持大文件
- 用户需对所在数据集具有 read 权限
- 响应包含正确的文件名（Content-Disposition）和 MIME 类型

**前端展示需求：**
- 数据项行提供下载按钮
- 点击后触发浏览器下载

### 3.6 系统数据清理（Prune）

**功能描述：** 完全重置系统数据，用于开发调试或重新初始化。

**详细需求：**
- **清理数据（prune_data）**：清除所有用户数据
- **清理系统（prune_system）**：可选择性清理：

| 选项 | 说明 | 默认值 |
|------|------|--------|
| graph | 图数据库 | 开启 |
| vector | 向量数据库 | 开启 |
| metadata | 关系型数据库元数据 | 关闭 |
| cache | 缓存 | 开启 |

- 此操作不可恢复

**前端展示需求：**
- 系统设置页提供「重置数据」按钮
- 提供多选项让用户选择清理范围
- 二次确认（输入确认文字）

## 4. 知识搜索

知识搜索是知识库的核心使用入口。系统提供多种搜索类型，覆盖从简单文本匹配到复杂知识推理的全部场景。所有搜索都支持权限控制和数据集范围限定。

### 4.1 搜索类型一览

| 搜索类型 | 名称 | 需要 LLM | 速度 | 适用场景 |
|----------|------|---------|------|---------|
| GRAPH_COMPLETION ⭐ | 图谱补全搜索 | 是 | 慢 | 复杂问答、分析、总结 |
| RAG_COMPLETION | RAG 补全搜索 | 是 | 中 | 文档检索、事实查找 |
| CHUNKS | 文本块搜索 | 否 | 快 | 查找段落、引用 |
| CHUNKS_LEXICAL | 词法搜索 | 否 | 快 | 精确关键词匹配 |
| SUMMARIES | 摘要搜索 | 否 | 快 | 快速概览、文档摘要 |
| CODE | 代码搜索 | 否 | 中 | 查找函数、类、代码模式 |
| CYPHER | Cypher 查询 | 否 | 快 | 精确图遍历、调试 |
| FEELING_LUCKY | 智能搜索 | 是 | 变化 | 通用查询 |
| TEMPORAL | 时序搜索 | — | — | 时间维度检索 |
| TRIPLET_COMPLETION | 三元组补全 | 是 | 中 | 三元组关系推理 |
| GRAPH_SUMMARY_COMPLETION | 图谱摘要补全 | 是 | 中 | 图谱级摘要问答 |
| NATURAL_LANGUAGE | 自然语言搜索 | — | — | 自然语言查询 |
| GRAPH_COMPLETION_COT | 思维链图谱补全 | 是 | 慢 | 深度推理 |
| GRAPH_COMPLETION_CONTEXT_EXTENSION | 上下文扩展 | 是 | 慢 | 扩展上下文搜索 |
| CODING_RULES | 编码规则搜索 | 否 | 快 | 查找编码规则（详见 M3） |

### 4.2 各搜索类型详细需求

#### 4.2.1 图谱补全搜索（GRAPH_COMPLETION）⭐ 推荐默认

- 搜索流程：查询 → 图谱上下文检索 → LLM 推理生成回答
- 返回 LLM 生成的对话式回答
- 适用场景：复杂问题分析、知识总结、深度洞察
- 支持自定义 system_prompt 控制回答风格

#### 4.2.2 RAG 补全搜索（RAG_COMPLETION）

- 搜索流程：查询 → 向量相似度检索 chunk → LLM 基于 chunk 生成回答
- 返回 LLM 生成的回答
- 适用场景：直接文档检索、特定事实查找
- 速度中等，无图谱遍历开销

#### 4.2.3 文本块搜索（CHUNKS）

- 搜索流程：查询 → 向量嵌入 → 相似度匹配
- 返回排序的文本块列表，包含源文档元数据
- 适用场景：查找特定段落、引用、精确内容
- 速度最快，不调用 LLM

#### 4.2.4 词法搜索（CHUNKS_LEXICAL）

- 基于 token 级别的词法匹配（如 Jaccard 相似度）
- 适用场景：精确关键词匹配、停用词感知查找
- 返回排序的文本块（可选带分数）

#### 4.2.5 摘要搜索（SUMMARIES）

- 返回在 Cognify 阶段预生成的内容摘要
- 支持多层级摘要（从详细到概览）
- 适用场景：快速概览、文档摘要、主题总结

#### 4.2.6 代码搜索（CODE）

- 针对代码文件的专用搜索
- 能理解函数、类、模块等代码结构
- 返回结构化的代码信息和上下文

#### 4.2.7 Cypher 查询（CYPHER）

- 用户直接输入 Cypher 查询语句
- 在图数据库上执行
- 返回原始查询结果

#### 4.2.8 智能搜索（FEELING_LUCKY）

- 分析查询意图，智能选择搜索模式
- 使用 LLM 判断最优搜索策略

### 4.3 搜索通用能力

**所有搜索类型共享以下通用功能：**

| 能力 | 参数 | 说明 |
|------|------|------|
| 数据集范围限定 | datasets / dataset_ids | 按数据集名称或 UUID 限定搜索范围 |
| 结果数量控制 | top_k | 控制返回结果数量（默认 10） |
| 自定义回答风格 | system_prompt | 控制 LLM 生成回答的风格和格式 |
| 节点集过滤 | node_name | 限定搜索到特定的 node_set 节点集 |
| 纯上下文模式 | only_context | 仅返回检索到的上下文，不调用 LLM |
| 详细模式 | verbose | 返回详细结果信息，包含图谱表示 |
| 搜索历史 | — | 自动保存搜索查询记录 |
| 交互保存 | save_interaction | 将查询-上下文-回答关系保存到图谱 |
| 会话管理 | session_id | 缓存 Q&A 交互 |
| 宽搜索 | wide_search_top_k | 宽搜索阶段的 top_k（默认 100） |
| 三元组距离惩罚 | triplet_distance_penalty | 图谱搜索距离惩罚系数（默认 3.5） |
| 检索器配置 | retriever_specific_config | 检索器专用配置 |
| 权限控制 | — | 搜索结果受数据集权限约束 |

### 4.4 搜索历史

**功能描述：** 查看用户的历史搜索记录。

**详细需求：**
- 按用户查询搜索历史
- 每条记录包含：
  - 搜索 ID
  - 搜索文本
  - 执行用户
  - 搜索时间

**前端展示需求：**
- 搜索页面提供：
  - 搜索输入框
  - 搜索类型下拉选择
  - 数据集范围选择（多选）
  - top_k 数量设置
  - 高级选项折叠面板（system_prompt、node_name 等）
- 搜索结果展示区：
  - 对话式搜索结果（GRAPH_COMPLETION、RAG_COMPLETION）
  - 列表式搜索结果（CHUNKS、SUMMARIES、CODE）
  - 原始数据展示（CYPHER）
- 搜索历史侧边栏
