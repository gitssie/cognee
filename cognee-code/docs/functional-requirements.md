# Cognee-Code 功能需求文档

> 基于 Cognee 项目构建的 AI 内存/记忆、知识库、编码规则、任务管理系统

---

## 一、项目定位

Cognee-Code 是一个基于 Cognee 引擎的 AI 辅助开发平台，为 AI Agent 提供持久化的**记忆管理**、**知识库管理**、**编码规则管理**和**任务管理**能力。系统依托 Cognee 的知识图谱引擎、向量搜索和 LLM 集成，为 AI 提供结构化的长期记忆和上下文感知能力。

---

## 二、功能模块总览

| 模块编号 | 模块名称 | 说明 |
|---------|----------|------|
| M1 | 知识库与文件管理 | 知识数据的添加、组织、检索、删除及文件生命周期管理 |
| M2 | Graph Knowledge（知识图谱） | 知识图谱的构建、查询、可视化、本体管理 |
| M3 | Code Rule 管理 | AI 编码规则的创建、存储、检索、关联 |
| M4 | AI Task 管理 | AI 任务的创建、追踪、执行状态管理 |

## 三、M1：知识库与文件管理（详细需求）

### 3.1 数据集（Dataset）管理

数据集是知识库的顶层组织单位。每个数据集归属于一个所有者用户，并可通过权限系统共享给其他用户。数据集内包含多个数据项（文件/文本），是知识处理（Cognify）和搜索的基本单元。

#### 3.1.1 创建数据集

**功能描述：** 用户可以创建一个新的知识数据集来组织不同主题或项目的知识数据。

**详细需求：**
- 用户通过名称创建数据集
- 若当前用户已拥有同名数据集，直接返回已有数据集，不重复创建
- 新创建的数据集自动授予当前用户全部权限：read（读取）、write（写入）、share（共享）、delete（删除）
- 数据集包含以下属性：
  - ID（UUID，自动生成）
  - 名称（用户指定）
  - 所有者 ID（创建用户）
  - 租户 ID（多租户隔离，可为空）
  - 创建时间
  - 更新时间
- 数据集创建时不包含任何数据，数据通过「添加数据」功能导入

**前端展示需求：**
- 提供创建数据集的表单界面（仅需输入名称）
- 创建成功后刷新数据集列表

#### 3.1.2 查看数据集列表

**功能描述：** 用户可以查看自己有权访问的所有数据集。

**详细需求：**
- 列出当前用户具有 read 权限的所有数据集
- 每个数据集展示信息：
  - 数据集名称
  - 数据集 ID
  - 创建时间
  - 最后更新时间
  - 所有者 ID
- 支持区分「我创建的」与「他人共享给我的」数据集

**前端展示需求：**
- 数据集列表页，以卡片或表格形式展示
- 展示数据集名称、创建时间、数据量概览
- 支持点击进入数据集详情

#### 3.1.3 删除数据集

**功能描述：** 用户可以删除整个数据集，包括其包含的所有数据项。

**详细需求：**
- 通过数据集 ID 执行删除
- 用户必须对该数据集具备 delete 权限
- 删除后该数据集及其下所有数据项被永久移除
- 若数据项仅属于该数据集，则一并从数据表中删除
- 若数据项同时属于其他数据集，则仅解除关联关系

**前端展示需求：**
- 删除前弹出确认对话框，提示「此操作不可恢复」
- 删除成功后从列表中移除

#### 3.1.4 查看数据集处理状态

**功能描述：** 用户可以查看数据集的知识图谱构建处理状态。

**详细需求：**
- 查询指定数据集在 cognify_pipeline 中的执行状态
- 支持批量查询多个数据集的状态
- 状态值包括：
  - **pending**：排队等待处理
  - **running**：正在处理中
  - **completed**：处理完成
  - **failed**：处理失败
- 用户需对目标数据集具有 read 权限

**前端展示需求：**
- 在数据集列表或详情页显示处理状态标签（颜色区分）
- 处理中的数据集显示进度指示
- 失败状态提供错误信息提示

#### 3.1.5 本地目录扫描发现数据集

**功能描述：** 扫描本地文件目录，自动发现可作为数据集的子目录。

**详细需求：**
- 输入一个目录路径
- 返回该目录下可被识别为数据集的子目录名称列表

### 3.2 数据（Data/文件）管理

数据（Data）是知识库中的基本内容单元，代表一个文件或一段文本。每个数据项关联到一个或多个数据集。系统在添加数据时会自动提取文件元信息、生成内容哈希用于去重，并存储到本地/云端文件系统。

#### 3.2.1 添加数据

**功能描述：** 用户向指定数据集中导入知识数据（文件、文本、URL 等）。

**支持的输入类型：**
- **文本字符串**：直接输入的文本内容
- **本地文件**：
  - 绝对路径：`/path/to/document.pdf`
  - File URL：`file:///path/to/document.pdf` 或 `file://relative/path.txt`
  - S3 路径：`s3://bucket-name/path/to/file.pdf`
- **文件上传**：通过表单上传的文件流（BinaryIO）
- **HTTP URL**：远程网页内容（需开启 ALLOW_HTTP_REQUESTS）
- **GitHub 仓库 URL**：自动克隆并处理整个仓库
- **批量添加**：以上类型的混合列表

**支持的文件格式：**
- 文本文件：`.txt`、`.md`、`.csv`
- PDF 文档：`.pdf`
- 图片文件：`.png`、`.jpg`、`.jpeg`（通过 OCR/视觉模型提取文本）
- 音频文件：`.mp3`、`.wav`（转录为文本）
- 代码文件：`.py`、`.js`、`.ts` 等（解析结构和内容）
- Office 文档：`.docx`、`.pptx`

**处理流程：**
1. **数据解析**：解析文件路径、验证可访问性
2. **内容提取**：从各种文件格式中提取文本内容
3. **去重检测**：基于内容哈希 + 用户 ID 生成唯一数据 ID，自动跳过已存在的相同内容
4. **数据集存储**：存储处理后的内容到指定数据集
5. **元数据记录**：记录文件名、扩展名、MIME 类型、文件大小、内容哈希、原始存储路径等
6. **权限分配**：为用户授予数据集的 read/write/delete/share 权限

**详细需求：**
- 数据必须归属到一个数据集（通过 datasetName 或 datasetId 指定）
- 支持 node_set 标签分组，用于知识图谱中的数据组织和访问控制
- 支持自定义标签（label）标注数据项
- 支持自定义 loader 引擎选择
- 支持增量加载模式（默认开启），避免重复处理相同内容
- 每个数据项包含以下属性：
  - ID（基于内容哈希+用户ID自动生成）
  - 名称（文件名或自动生成）
  - 标签（用户自定义）
  - 扩展名、MIME 类型（存储文件的）
  - 原始扩展名、原始 MIME 类型（原始文件的）
  - 原始数据存储位置
  - 内容哈希（用于去重）
  - 所有者 ID、租户 ID
  - node_set（节点集标签）
  - pipeline_status（管道处理状态）
  - token_count（文本 token 数）
  - 文件大小（字节）
  - 创建时间、更新时间、最后访问时间
  - loader_engine（使用的加载引擎名称）
  - 外部元数据（JSON，可附加自定义信息）

**前端展示需求：**
- 提供文件拖拽上传区域
- 提供 URL 输入框（支持 HTTP URL、GitHub URL）
- 提供文本直接输入框
- 上传时可选择或创建目标数据集
- 上传时可设置 node_set 标签
- 展示上传进度和结果

#### 3.2.2 查看数据列表

**功能描述：** 查看某个数据集下的所有数据项（文件/文本）。

**详细需求：**
- 按数据集 ID 查询其下所有数据项
- 需对该数据集具有 read 权限
- 每个数据项展示信息：
  - 数据 ID
  - 名称
  - 标签
  - 扩展名
  - MIME 类型
  - 原始文件存储位置
  - 所属数据集 ID
  - 创建时间
  - 更新时间

**前端展示需求：**
- 数据集详情页内展示数据列表（表格形式）
- 支持按名称、类型、时间排序
- 支持文件类型图标展示
- 每行提供操作按钮（下载、删除、更新）

#### 3.2.3 更新数据

**功能描述：** 更新已有数据项的内容（替换文件），自动重新构建知识图谱。

**详细需求：**
- 输入参数：目标数据 ID + 目标数据集 ID + 新数据内容
- 更新流程（原子操作）：
  1. 删除旧数据及其在图谱中的子图
  2. 添加新数据到数据集
  3. 自动触发 Cognify 重建知识图谱
- 支持的输入类型与「添加数据」一致
- 支持 node_set 标签重新设置
- 返回更新后的管道运行信息

**前端展示需求：**
- 数据项详情页提供「替换文件」按钮
- 显示更新进度（因为会触发 Cognify）

#### 3.2.4 删除数据

**功能描述：** 从数据集中删除特定的数据项。

**详细需求（两种入口）：**

**入口一：按数据 ID 删除（支持软/硬模式）**
- 输入参数：data_id + dataset_id + mode
- 用户需对数据集具有 delete 权限
- 删除流程：
  1. **图谱子图删除**（按固定顺序维护图完整性）：
     - 删除孤立实体
     - 删除孤立实体类型
     - 删除 made_from 节点
     - 删除文档 chunk
     - 删除文档节点
  2. **向量库删除**：从所有向量集合中删除对应的嵌入向量
  3. **关系账本更新**：标记 graph_relationship_ledger 中相关记录的 deleted_at
  4. **关系型数据库删除**：
     - 解除数据与数据集的关联
     - 若数据不再属于任何数据集，则删除数据记录
- **软删除（soft）**：仅删除文档直接相关节点，保留可能被其他文档共享的实体节点
- **硬删除（hard）**：额外删除度为 1 的孤立 Entity 和 EntityType 节点（不再被任何其他文档引用的实体）
- 返回删除结果：已删除节点数量、已删除节点 ID 列表

**入口二：从数据集中移除数据项**
- 输入参数：dataset_id + data_id
- 仅解除数据与数据集的关联
- 不执行图谱级别的删除

**前端展示需求：**
- 数据项行提供删除按钮
- 删除前弹出确认对话框
- 提供软删除/硬删除选项说明
- 删除后刷新列表

#### 3.2.5 下载原始数据

**功能描述：** 下载数据项的原始文件。

**详细需求：**
- 支持两种存储来源：
  - **本地文件系统**：直接通过 FileResponse 返回文件
  - **S3 存储**：通过 StreamingResponse 流式下载，支持大文件
- 用户需对所在数据集具有 read 权限
- 响应包含正确的文件名和 MIME 类型

**前端展示需求：**
- 数据项行提供下载按钮
- 点击后触发浏览器下载

#### 3.2.6 系统数据清理（Prune）

**功能描述：** 完全重置系统数据，用于开发调试或重新初始化。

**详细需求：**
- **清理数据（prune_data）**：清除所有用户数据
- **清理系统（prune_system）**：可选择性清理：
  - graph（图数据库）
  - vector（向量数据库）
  - metadata（关系型数据库元数据）
  - cache（缓存）
- 此操作不可恢复

**前端展示需求：**
- 系统设置页提供「重置数据」按钮
- 提供多选项让用户选择清理范围
- 二次确认（输入确认文字）

### 3.3 知识搜索

知识搜索是知识库的核心使用入口。系统提供 13 种搜索类型，覆盖从简单文本匹配到复杂知识推理的全部场景。所有搜索都支持权限控制和数据集范围限定。

#### 3.3.1 图谱补全搜索（GRAPH_COMPLETION）⭐ 推荐默认

**功能描述：** 基于知识图谱上下文 + LLM 的自然语言问答。

**详细需求：**
- 搜索流程：查询 → 图谱上下文检索 → LLM 推理生成回答
- 返回 LLM 生成的对话式回答
- 适用场景：复杂问题分析、知识总结、深度洞察
- 支持自定义 system_prompt 控制回答风格

**前端展示需求：**
- 对话式 UI，展示 AI 生成的回答
- 可选展示引用的图谱上下文

#### 3.3.2 RAG 补全搜索（RAG_COMPLETION）

**功能描述：** 传统 RAG 问答，基于文档 chunk 而不使用图谱结构。

**详细需求：**
- 搜索流程：查询 → 向量相似度检索 chunk → LLM 基于 chunk 生成回答
- 返回 LLM 生成的回答
- 适用场景：直接文档检索、特定事实查找
- 速度中等，无图谱遍历开销

#### 3.3.3 文本块搜索（CHUNKS）

**功能描述：** 纯语义向量相似度搜索，返回原始文本片段。

**详细需求：**
- 搜索流程：查询 → 向量嵌入 → 相似度匹配
- 返回排序的文本块列表，包含源文档元数据
- 适用场景：查找特定段落、引用、精确内容
- 速度最快，不调用 LLM

#### 3.3.4 词法搜索（CHUNKS_LEXICAL）

**功能描述：** 基于词法匹配（如 Jaccard 相似度）的文本块搜索。

**详细需求：**
- 基于 token 级别的词法匹配
- 适用场景：精确关键词匹配、停用词感知查找
- 返回排序的文本块（可选带分数）

#### 3.3.5 摘要搜索（SUMMARIES）

**功能描述：** 返回预生成的分层摘要。

**详细需求：**
- 返回在 Cognify 阶段预生成的内容摘要
- 支持多层级摘要（从详细到概览）
- 适用场景：快速概览、文档摘要、主题总结
- 速度快，无实时计算

#### 3.3.6 代码搜索（CODE）

**功能描述：** 代码专用搜索，理解代码语法和语义。

**详细需求：**
- 针对代码文件的专用搜索
- 能理解函数、类、模块等代码结构
- 适用场景：查找函数实现、类定义、代码模式
- 返回结构化的代码信息和上下文

#### 3.3.7 Cypher 查询（CYPHER）

**功能描述：** 直接对图数据库执行 Cypher 查询语句。

**详细需求：**
- 用户输入 Cypher 查询语句
- 直接在图数据库上执行
- 返回原始查询结果
- 适用场景：高级用户、精确图遍历、调试

#### 3.3.8 智能搜索（FEELING_LUCKY）

**功能描述：** 系统自动选择最合适的搜索类型并执行。

**详细需求：**
- 分析查询意图，智能选择搜索模式
- 使用 LLM 判断最优搜索策略
- 适用场景：用户不确定使用哪种搜索时

#### 3.3.9 时序搜索（TEMPORAL）

**功能描述：** 基于时间维度的知识检索。

**详细需求：**
- 支持时间感知的知识检索
- 适用场景：查找特定时间段的信息、时序关系分析

#### 3.3.10 其他搜索类型

- **TRIPLET_COMPLETION**：基于三元组的补全搜索
- **GRAPH_SUMMARY_COMPLETION**：图谱摘要补全搜索
- **NATURAL_LANGUAGE**：自然语言搜索
- **GRAPH_COMPLETION_COT**：思维链图谱补全搜索
- **GRAPH_COMPLETION_CONTEXT_EXTENSION**：上下文扩展图谱补全搜索
- **CODING_RULES**：编码规则搜索（详见 M3 模块）

#### 3.3.11 搜索通用能力

**所有搜索类型共享以下通用功能：**
- **数据集范围限定**：可按数据集名称或 UUID 限定搜索范围
- **结果数量控制**：通过 top_k 参数控制返回结果数量（默认 10，最大 100）
- **自定义 system_prompt**：控制 LLM 生成回答的风格和格式
- **node_name 过滤**：限定搜索到特定的 node_set 节点集
- **only_context 模式**：仅返回检索到的上下文，不调用 LLM 生成回答（调试/验证用）
- **verbose 模式**：返回详细的结果信息，包含图谱表示
- **搜索历史记录**：自动保存搜索查询记录，支持查看历史搜索
- **交互保存**：可选将查询-上下文-回答关系保存到图谱中
- **会话管理**：支持 session_id 用于缓存 Q&A 交互
- **权限控制**：搜索结果受数据集权限约束，无权限则返回空结果

**前端展示需求：**
- 搜索页面提供：
  - 搜索输入框
  - 搜索类型下拉选择
  - 数据集范围选择（多选）
  - top_k 数量设置
  - 高级选项折叠面板（system_prompt、node_name 等）
- 搜索结果展示区：
  - 对话式搜索结果展示（GRAPH_COMPLETION、RAG_COMPLETION）
  - 列表式搜索结果展示（CHUNKS、SUMMARIES、CODE）
  - 原始数据展示（CYPHER）
- 搜索历史侧边栏

## 四、M2：Graph Knowledge（知识图谱）

### 4.1 知识图谱构建（Cognify）

#### 4.1.1 数据处理管道
- 将已添加的原始数据转化为结构化知识图谱
- 处理流程：文档分类 → 文本分块 → 实体抽取 → 关系检测 → 图谱构建 → 向量嵌入生成 → 内容摘要

#### 4.1.2 自定义提示词
- 支持自定义 prompt 控制实体抽取和图谱生成的方式
- 可指定关注特定类型的实体和关系

#### 4.1.3 本体（Ontology）引导
- 支持上传 OWL 格式本体文件
- 构建知识图谱时可引用已上传的本体来约束图谱结构
- 支持多本体组合使用

#### 4.1.4 后台异步执行
- 支持 Cognify 管道后台运行（不阻塞请求）
- 提供 WebSocket 订阅实时获取管道运行状态

#### 4.1.5 管道状态查询
- 查询 Cognify 管道的执行进度和状态

#### 4.1.6 批次处理控制
- 支持设置每批处理的 chunk 数量（chunks_per_batch）

### 4.2 本体管理

#### 4.2.1 上传本体
- 支持上传 OWL 格式本体文件
- 以用户自定义 key 标识
- 可附加描述信息

#### 4.2.2 查看本体列表
- 列出当前用户已上传的所有本体
- 展示本体的 key、文件名、大小、上传时间

### 4.3 知识图谱可视化

#### 4.3.1 数据集图谱可视化
- 按数据集 ID 获取知识图谱的可视化数据（节点和边）
- 返回交互式 HTML 可视化页面

#### 4.3.2 图谱数据获取
- 获取数据集图谱的结构化数据（节点列表 + 边列表）
- 节点包含：ID、标签、属性
- 边包含：源节点、目标节点、关系标签

### 4.4 图谱富化（Memify）

#### 4.4.1 自定义管道富化
- 对已构建的知识图谱执行自定义抽取和富化任务
- 支持自定义 extraction_tasks 和 enrichment_tasks
- 可提供额外数据作为输入，也可基于已有图谱数据进行处理
- 支持 node_name 过滤处理特定子图

### 4.5 数据同步

#### 4.5.1 同步到云端
- 将本地数据集同步到 Cognee Cloud
- 支持按数据集 ID 选择性同步
- 后台异步执行，支持自动重试

## 五、M3：Code Rule 管理（编码规则）

### 5.1 编码规则创建

#### 5.1.1 从对话中自动提取规则
- 分析用户-AI 交互记录，自动提取编码最佳实践和规则
- 基于 LLM 进行规则提取（使用专用 prompt 模板）
- 提取结果存储为 Rule 数据点，关联到 coding_agent_rules 节点集

#### 5.1.2 手动写入规则/记忆
- 通过 write_memory 工具手动存入编码规则或重要记忆
- 支持自由文本格式，如：用户偏好、项目约束、设计决策、最佳实践等

### 5.2 编码规则检索

#### 5.2.1 搜索编码规则（CODING_RULES 搜索类型）
- 通过 CODING_RULES 搜索类型查询已存储的编码规则
- 基于语义相似度匹配最相关的规则

#### 5.2.2 读取全部记忆
- 通过 read_memory 获取所有已存储的编码规则/记忆
- 返回完整规则列表，按序号编排

### 5.3 规则关联

#### 5.3.1 规则与源文档关联
- 自动将提取的规则关联到原始文档 chunk
- 建立 rule_associated_from 关系，实现规则溯源

#### 5.3.2 去重与增量更新
- 提取规则时参考已有规则，避免重复
- 已有规则作为上下文传入 LLM，实现增量式规则积累

## 六、M4：AI Task 管理

### 6.1 交互管理

#### 6.1.1 保存用户-AI 交互
- 记录用户查询和 AI 回答的完整交互
- 交互数据通过 Cognify 管道处理并纳入知识图谱
- 标记为 user_agent_interaction 节点集

#### 6.1.2 OpenAI 兼容响应

- 内置 search 和 cognify 两个默认工具函数

### 6.2 管道任务管理

#### 6.2.1 Cognify 管道状态追踪
- 查询指定数据集的 Cognify 管道执行状态
- 获取管道运行信息：进度、执行时间、完成状态

#### 6.2.2 WebSocket 实时订阅
- 通过 WebSocket 连接实时获取管道运行事件
- 支持按 pipeline_run_id 订阅特定运行

### 6.3 Notebook 管理（交互式代码运行）

#### 6.3.1 Notebook CRUD
- 创建 Notebook（包含名称和单元格列表）
- 查看用户所有 Notebook
- 更新 Notebook 名称和单元格内容
- 删除 Notebook

#### 6.3.2 执行 Notebook 单元格
- 在本地沙箱中执行指定单元格的代码
- 返回执行结果和错误信息
