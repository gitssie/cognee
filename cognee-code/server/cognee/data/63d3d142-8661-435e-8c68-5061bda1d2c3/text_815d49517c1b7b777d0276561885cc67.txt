# 需求整理

## 对象关联功能

**设计**
1. 提供一个新增关联对象的便捷操作入口，通过将关联字段映射，在新增关联对象的时候将主体字段的值填充到关联对象中，减少用户录入操作。
2. 关联对象随主体数据更新的时候同步更新。比如新增产品的时候，自动将产品同步更新到关联的价格表明细中。减少对用户频繁的去维护价格表。

#### 字段
 | 字段名称 | 字段标识 | 字段类型  | 说明 | 
 |---------|---------|-------| ---------|
 | 编号 | memberNo | 多行文本类型 | 
 | 源对象 | source | 关联值   | 
 | 目标对象 | target | 关联值  | 
 | 关联类型 | type | 单选类型  | 
 | 执行动作 | action | 单选类型  | 
 | 执行脚本 | script | JSON  | 
 | 字段映射 | memberPosition | JSON  |

### 工作报告(workReport)
1. 业务类型:日报,周报,月报.
2. 快捷过滤包含我提交的,提交给我,抄送给我的,我收藏的,支持分屏看板
3. 状态:
    > 草稿: 保存为草稿
    > 提交: 提交报告,批阅人可批阅, 已提交批阅的不可编辑,可撤回再编辑. 
    > 已阅: 批阅人已批阅
4. 提交期限: 时间超过期限值不可提交.
    >日报: 17:00 
    >周报: 周五:17:00
    >月报: 本月31号 17:00

#### 功能设计
1. 多类型报告：支持日报、周报、月报三种类型，每种类型有独立的提交截止时间
2. 模板管理：支持为不同业务类型定义报告模板，简化报告编写
3. 富文本编辑：支持富文本编辑器，允许格式化文本和插入图片
4. 草稿保存：支持随时保存为草稿，避免数据丢失
5. 版本管理：支持报告撤回、重新编辑功能
6. 批阅流程：支持配置批阅人，批阅人可添加评论、评分、建议
7. 智能提醒：自动在提交截止时间前进行提醒，超期后禁止提交
8. 报告统计：支持统计报告提交率、超期率等数据
9. 快捷过滤：支持按"我的"、"给我的"、"抄送给我"、"我收藏的"等快速过滤
10. 分屏看板：支持分屏看板展示，便于审阅多个报告

## 销售线索（salesLead）
### 业务关联
1. **市场活动**：销售线索可来自市场活动，记录活动来源
2. **销售机会**：销售线索可转换为销售机会
3. **客户**：销售线索转换后成为客户

### 问题点
1. 区域公海状态： 怎么触发回归公海池
2. 线索创建之后是否进入公海？
3. 线索转换的业务逻辑是怎么样的？
4. 回收的业务逻辑？？

### 功能设计
1. 线索获取渠道多元化：支持从市场活动、企微、微信、手工录入等多种渠道获取线索
2. 线索转换：根据该线索，将线索转换为客户信息并添加联系人信息，亦可在此同时建立销售机会，线索转换之后线索记录删除掉，客户记录新增
3. 线索转移：将该线索转移给他人负责，手动指定负责人
4. 业务类型编辑：将该线索的业务类型改变，后续的视图将根据业务类型变更
5. 线索回收：支持对长期未跟进的线索进行回收管理
6. 公海池管理：支持线索的所属人移除，改变公海池状态，退回到公海池中
7. 跟进状态管理：支持按跟进状态（待跟进、进行中、已转化、已关闭）进行分类管理
8. 地理位置管理：记录线索所在地区（省市区），便于地域化管理
9. 企微集成：支持从企业微信集成线索数据（好友、标签、头像等）
10. 线索评分：支持对线索进行评分，帮助销售人员优先级管理

10. 公海管理：支持区域公海状态管理，支持设置预计回收时间
11. 查重管理：支持标记疑似重复机会，防止数据重复

### 功能设计
1. 多业务类型：支持销售框架合同、租赁合同等多种业务类型
2. 合同分类：支持产品销售、服务、业务合作、代理分销等多种合同类型
3. 生命周期管理：支持执行中、结束、意外终止等状态管理
4. 付款方式：支持现金、支票、银行转账、分期等多种付款方式

11. 开票管理：统计已开票金额和未开票金额
12. 收款计划：支持制定和管理收款计划

3. 取消生效：建立了收款单据之后不可取消
4. 收款计划：
6. 交付记录：根据订单自动带出 客户，交付订单，收件/联系信息，交付明细（列表展现）；
7. 资产： 基本信息（主要为产品信息），位置信息，设备二维码，其他信息
    >订单关联资产的业务逻辑没太理解，业务场景是什么？
8. 派工单：
9. 退款： 进入收款单里面操作退款和建立退款单
10. 发票：根据订单和应收单创建发票
    > 开票金额不能大于应收总额
11. 业务类型: 订单,退货单

#### 功能设计补充
1. 订单创建流程：分步建立订单，包括资料填写（基本信息、收货/联系信息、金额信息、其他信息）→ 创建订单（选择价格表、添加产品、设置数量/价格/折扣/交货日期）→ 提交审核
2. 订单审批：支持配置订单审批流程，审批人可审批、驳回或通过
3. 生效管理：改变订单状态为"已生效"，自动生成订单应收单和应收单明细；确认生效后在合同详情-发票收款页面可看到应收信息
4. 取消生效：支持取消订单生效，但建立了收款单据之后不可取消
5. 产品明细：支持添加多个产品，配置单价、数量、折扣、交货日期等信息，自动计算金额
6. 收款计划：支持配置收款计划和收款方式
7. 交付管理：根据订单自动带出客户、交付地址、联系信息，支持记录交付明细和交付记录
8. 资产管理：将产品关联为资产，记录位置信息、设备二维码等，便于后续资产追踪
9. 派工管理：支持生成派工单，分配派工任务
10. 退款管理：在收款单中支持操作退款和建立退款单
11. 发票管理：根据订单和应收单创建发票，开票金额不能大于应收总额
12. 多业务类型：支持订单类型（正常订单、退货单等）切换
13. 版本管理：记录订单版本号，支持订单变更管理
14. 折扣管理：支持系统折扣和额外折扣，自动计算总折扣
15. 返利管理：支持返利兑换账户管理
16. 金额统计：支持原价、折扣后价格、总金额等多维度金额统计
17. 订单关联：支持关联销售机会、合同等业务对象
18. 订单状态追踪：从创建→审批→生效→交付→完成的全生命周期管理

### 交付记录（deliveryRecord）
#### 功能设计
1. 交付流程：新建-发起收货确认-
2. 物流信息：根据运单号调用物流接口显示物流进度信息
3. 打印： 发货清单打印（pdf/excel）

### 领料单(materialRequisition)
#### 功能设计
1. 关联仓库和产品，领料明细必须从选择的仓库中选择产品。
2. 确认领料：确认领料之后，自动生成个人库入库单和出库单。
    > 个人库出入库记录，记录为入库单。
    > 出库单的状体为出库
3. 撤销领料：确认领料之后可撤销领料，撤销领料将把确认领料生成的生成个人库入库单和出库单删除
4. 拒绝领料：拒绝领料需备注拒绝原因，并将领料单状态变更为“已拒绝”，领料流程结束。
### 退料单
#### 功能设计
1. 关联仓库和产品，领料明细必须从选择的仓库中选择产品。
2. 确认退料：确认退料之后，自动生成个人库入库单和入库单。
    > 个人库出入库记录，记录为出库单。对应领料的入库单
    > 入库单的状态为入库，与领料单的出款单对应
3. 撤销退料：确认退料之后可撤销领料，撤销退料将把确认退料生成的生成个人库出库单和如库单删除

### 区域模型(regionalModel)
区域模型分为普通区域模型和公海区域模型不同业务类型
1. 区域数据，关联数据实体对象。包含启用，创建后分配，编辑后分配，分配到负责人和所属区域字段（该项针对普通区域）。
   数据跟随，设置该实体关联的实体后续的分配跟随该主体实体。
2. 实体分配条件，对关联的实体进行条件设定，条件包含（实体，分配方式，帅选条件，移除团队成员），设置之后可以进行预分配。预分配之后生成分配记录。也可以设置预览看板，分配完之后通过看板查看数据。

公海池类型特有功能，针对关联的实体一一设定：
1.分组范围
公海池成员领取范围
 本区域
 本区域 + 一级上级区域
 本区域 + 所有上级区域
 本区域 + 一级下级区域
 本区域 + 所有下级区域
2.例外规则：
 自建/其他规则。其他规则根据实体字段设定条件。
3.全局个人私池
  私池限制设定，以及特殊人员设置（可选择人员设置超过规定的限额私池数量）
4.回收设置
  配置回收的规则和回收提醒配置
5.原因设置
延期原因、详情
退回原因、详情
6.公海字段
配置公海池显示对应的实体哪些字段可在页面展示。
7.通用设置
通知设置：分配通知，手动回收通知，自动回收提前提醒，自动回收通知

### 区域(regional)
1. 区域支持上下级
2. 每个区域有独立的成员配置，包含部门和员工。可配置权限（实体，权限[只读，修改，无权限],区域权限范围[本级，本机及下级]）及操作的数据实体范围。
3. 数据分配规则，包含维度混合条件，包含客户，不包含客户。维度配置在系统设置里面设置（主子明细层级），维度配之后需关联对应的实体。用于自动回收和手动回收，用于每条区域记录对数据进行过滤。
4. 公海类型区域可以设置通用回收区域，手动回收或自动回收时，可以指定回收到一个特定公海池分组（只有开启了公共回收的公海池分组才能被选择）。 设置了通用回收区域的区域记录不能进行数据分配规则，因为他是公共的。

公海池区域分组特有功能：
1. 领取设置：是否手动领取，个人领取数量设定，是否允许“非”本组成员转入记录到本组，是否允许本组成员将数据转出到其他组，是否允许跨组领取本组数据记录
2. 分配设置：是否允许分组管理员超私池上限分配分组数据
3. 退回设置：是否允许退回，退回之后多少天之内不能被重复认领，手动退回超过多少次自动冻结包括（用户手动退回，自动回收，管理员手动回收），冻结多少天之后恢复。
4. 回收设置：自动回收配置（顺序，名称，起止时间，计时天数，收回条件，回收区域）；
   手动回收：是否允许分组管理员手动回收自建数据
5. 显示设置：配置实体哪些字段可以在页面显示
