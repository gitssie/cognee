# 关联查询去重分页最佳实践

## 问题场景

从关联表查询主表数据，且需要去重和分页：

- 从 `RegionalMembers` 查询不重复的 `User` 列表
- 从 `OrderItems` 查询不重复的 `Product` 列表

**核心问题：** 关联表有重复外键，需要去重后分页查询主表数据。

## Ebean Query API 核心知识

### 查询构建方式

```java
// 1. 使用 ebeanApi.createQuery() 创建查询
Query<RegionalMembers> query = ebeanApi.createQuery(RegionalMembers.class);

// 2. select() - 指定要查询的属性
query.select("user.id");  // 只查询 user.id 字段

// 3. setDistinct() - 去重查询
query.setDistinct(true);  // SQL: SELECT DISTINCT user_id

// 4. where() - 添加条件
query.where()
     .eq("regional.id", regionalId)
     .eq("status", 1);

// 5. orderBy() - 排序
query.orderBy("user.name asc");

// 6. setPaging() - 分页
query.setPaging(Paging.of(pageNumber, pageSize));  // 0-based page index

// 7. 执行查询
Set<Long> ids = query.findSingleAttributeSet();  // 单字段去重查询
List<RegionalMembers> list = query.findList();    // 查询实体列表
```

### 关键方法说明

| 方法                          | 说明             | SQL 示例                |
| ----------------------------- | ---------------- | ----------------------- |
| `select("user.id")`           | 选择特定字段     | `SELECT user_id`        |
| `setDistinct(true)`           | 去重查询         | `SELECT DISTINCT`       |
| `setPaging(Paging.of(0, 10))` | 分页（0 基索引） | `LIMIT 10 OFFSET 0`     |
| `where().eq()`                | 等值条件         | `WHERE field = ?`       |
| `where().in()`                | IN 条件          | `WHERE field IN (?, ?)` |
| `orderBy("field asc")`        | 排序             | `ORDER BY field ASC`    |
| `findSingleAttributeSet()`    | 单字段去重结果   | 返回 `Set<T>`           |
| `findList()`                  | 实体列表         | 返回 `List<T>`          |

## 错误实现 ❌

```java
// ❌ 直接查询关联表
BeanType<RegionalMembers> desc = ebeanApi.desc(RegionalMembers.class);
return dataService.queryMap(desc, model, queryForm.getQuery(), queryForm.getPageable());
```

**问题：** 返回 RegionalMembers（有重复），前端需要复杂的数据转换。

## 正确实现 ✅

### 后端 Controller

```java
@PostMapping("/api/data/object/regional/members")
@Query(apiKey = "regional", funcName = "MembersSelect")
@PreAuthorize("hasPermission('regional', 'list')")
public Either<Code, Slice<Map<String, Object>>> queryMembersSelect(
        Authentication authentication,
        Model model,
        @Validated @RequestBody QueryForm queryForm) {

    Pageable pageable = queryForm.getPageable();

    // 1. 解析查询条件
    Either<Code, Expression> whereE = ebeanApi.parsePredicate(
        ebeanApi.desc(RegionalMembers.class),
        queryForm
    );
    if (whereE.isLeft()) {
        return (Either) whereE;
    }

    // 2. 数据库层面去重+分页（关键）
    var query = ebeanApi.createQuery(RegionalMembers.class)
            .select("user.id")
            .setDistinct(true)  // 去重
            .setPaging(Paging.of(pageable.getPageNumber(), pageable.getPageSize()));  // 分页

    if (whereE.get() != null) {
        query.where().add(whereE.get());
    }

    Set<Long> distinctUserIds = query.findSingleAttributeSet();

    // 3. 空结果处理
    if (distinctUserIds.isEmpty()) {
        return Either.right(new SliceImpl<>(Collections.emptyList()));
    }

    boolean readEnd = distinctUserIds.size() < pageable.getPageSize();

    // 4. 批量查询 User
    List<User> users = ebeanApi.list(User.class, ebeanApi.expr().idIn(distinctUserIds));

    // 5. 使用 JS 函数配置转换（重要）
    Slice<Map<String, Object>> page = ebeanApi.toJSON(
        model,                          // JS 函数配置
        ebeanApi.desc(User.class),      // User 实体
        new SliceImpl<>(users, pageable, readEnd)
    );

    return Either.right(page);
}
```

## 关键点

1. **数据库层面去重+分页**：`setDistinct(true)` + `setPaging()`
2. **批量查询避免 N+1**：`ebeanApi.list(User.class, idIn(ids))`
3. **返回主表实体**：返回 User 而不是 RegionalMembers
4. **JS 函数配置字段**：`ebeanApi.toJSON(model, userDesc, slice)`
5. **使用 Slice 而不是 Page**：性能更好，不需要 COUNT 查询

## 性能优化

| 方式                | 查询次数 | 内存占用 | 去重   |
| ------------------- | -------- | -------- | ------ |
| 直接查询关联表      | 1        | 中       | ❌     |
| **数据库去重+分页** | **2**    | **低**   | **✅** |

## 扩展场景

### 多条件过滤

```java
var query = ebeanApi.createQuery(RegionalMembers.class)
        .select("user.id")
        .setDistinct(true)
        .where()
            .eq("regional.id", regionalId)
            .eq("status", 1)
            .ilike("user.name", "%张%")
        .setPaging(Paging.of(pageNumber, pageSize));
```

### 排序

```java
var query = ebeanApi.createQuery(RegionalMembers.class)
        .select("user.id")
        .setDistinct(true)
        .orderBy("user.name asc")
        .setPaging(Paging.of(pageNumber, pageSize));
```

## 总结

### Ebean Query API 使用要点

1. **select() 选择字段**：只查询需要的字段，减少数据传输
2. **setDistinct() 去重**：数据库层面去重，返回 Set 自动去重
3. **setPaging() 分页**：0-based page index，在数据库层面分页
4. **关联查询路径**：使用点号表示关联（如 `"user.id"`, `"customer.name"`）
5. **findSingleAttributeSet()**：单字段去重查询，性能最优

### 核心步骤

1. 在关联表上执行 `setDistinct(true)` + `setPaging()` 查询 ID
2. 使用 `idIn()` 批量查询主表数据
3. 使用 `ebeanApi.toJSON(model, desc, slice)` 应用 JS 配置
4. 返回 `Slice<Map<String, Object>>`

**性能关键：**

- 去重和分页在数据库层面完成
- 只查询当前页需要的数据
- 批量查询避免 N+1 问题
